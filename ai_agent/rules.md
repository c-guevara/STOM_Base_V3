# 코드 구현 및 수정 규칙
##### 1단계: 계획 수립
- 작업을 요청하면 먼저 간단한 초기계획 작성을 준비한다.
- ai_agent/working/plan_completed 폴더의 파일명을 검색한다.
- 검색한 파일명에 유사한 계획이 있으면 참고하여 초기계획을 작성한다.
- 초기계획 작성 중, 선택사항이 있으면 사용자에게 결정을 물어본다.
- 선택사항은 마우스로 쉽게 클릭해서 결정할 수 있게 반드시 선택메뉴 형태로 보여준다.
- 선택사항에는 반드시 추천의 정도와 간단한 설명을 덧붙인다.
- 초기계획을 보고 서로 상의해서 구체적인 계획으로 수정한다.
- 계획 상의 중, 새로운 아이디어가 있으면 선택 및 변경을 추천하고 제안한다.
- 계획작성에 참고할 사이트가 있으면 링크를 제공하여 결정에 도움을 제공한다.
- 구체적인 계획에는 반드시 코드의 간단 예시를 포함한다. 
- 수정된 계획을 보여줄때 마다 계획수립 완성도를 퍼센트로 표시해서 보여준다.
- 구체적인 계획작성이 완료되면 코드구현 및 추가수정제안 선택메뉴를 보여준다.
- 코드구현을 선택하기 전에는 절대 코드수정 및 추가작업을 하지 않는다.

##### 2단계: 코드구현 사전작업
- 코드구현 명령이 내려졌을때 다음의 사전 작업을 시작한다.
- ai_agent/working 폴더에 한글계획명_일자.txt 형태로 계획을 저장한다.
- 100줄 이하의 단순한 작업은 현재의 브렌치에서 바로 작업한다.
- 100줄 이상의 복잡한 작업은 새로운 브렌치를 생성하고 체크아웃하여 작업한다.

##### 3단계: 코드 구현
- 코드구현 사전작업이 완료되면 비로소 코드 구현을 시작한다.
- 코드는 항상 간결하고 깔끔하게 작성하며 속도우선을 고려하여 구현한다.
- 구현 시 클래스 또는 함수마다 간단한 설명을 주석으로 추가한다.
- 주석은 구글스타일 및 한글로 작성하고 타입표시와 빈줄을 포함시키지 않는다. 
- 코드 수정 및 작성 시 파일 단위별로 수정 후 간단한 문법 테스트를 하고 다음 작업을 진행한다. 
- 코드구현이 완료되면 작성한 계획을 다시 읽어보고 누락된 부분이 있으면 추가 구현한다.
- 누락없이 코드구현이 완료될때까지 계획 다시 읽기와 추가 구현을 반복한다.

##### 4단계: 코드 테스트
- 계획대로 코드가 구현완료되고 실행 테스트가 필요하면 별도의 테스트용 파일을 만든다.
- 테스트용 파일에는 반드시 로그 기록 코드를 포함시키고 ai_agent/working/log 폴더에 log_계획명_일자.txt파일로 기록한다.
- 테스트 시 오류가 발생하면 로그파일을 자세히 검토하여 다시 코드를 수정한다.
- 오류 없이 테스트가 완료될때까지 오류검토와 코드수정을 반복한다.

##### 5단계: 정리 및 마감
- 오류 테스트를 통과하여 코드구현이 완료되면 2단계에서 작성한 계획파일 다시 읽어본다.
- 계획 단계에서는 예상하지 못했던 어려운 난관이나 여러차례 수정했던 부분을 계획파일에 자세히 추가 기록한다.
- 추가 기록한 계획파일을 ai_agent/working/plan_completed 폴더로 이동시킨다.
- 4단계에서 테스트용으로 생성한 파일이 있으면 삭제한다.
- 계획파일과 로그파일은 기록용으로 남겨두기 위해 삭제하지 않는다.
- 구현된 코드를 작업한 브렌치에 커밋한다.
- 자세한 완료보고서를 보여주고 작업을 마감한다.

# 전략 생성 규칙
- ai_agent/strategy.txt 파일에서 변수설명과 예제를 자세히 읽어본다.
- 변수설명과 예제를 바탕으로 초기계획을 작성하여 제출한다.
- 초기계획 제출 이후의 과정은 코드 구현 및 수정 규칙을 준수한다.
- 데이터 형태는 1초스냅샷과 1분봉 두가지이며 동시 사용이 불가능하다.
- 매수전략 작성 시 반드시 매수 = True 상태임을 유념하고 전략을 작성한다.
- 매도전략 작성 시 반드시 매도 = False 상태임을 유념하고 전략을 작성한다.
- 매수전략과 매도전략을 분리하여 작성한다.
- 생성한 전략은 ai_agent/strategy 폴더에 한글전략명_생성일자및시간.txt파일 형태로 저장한다.
